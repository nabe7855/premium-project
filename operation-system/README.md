# SB-Operation System Master Plan (v1.0)

このフォルダは、AI（Grok / Perplexity / GPT-4o）を活用して、低コストで集客と採用を最大化するための運用システム一式を管理しています。

## 1. システム概要
「トレンドリサーチ」から「コンテンツ生成」「SNS配信」「効果分析」までの一巡を、人間が最小限の労力（確認のみ）で行える体制を構築します。

### **【運用サイクル】**
1.  **リサーチ**: Grok (Xの生の声) ＆ Perplexity (ウェブ全域) からトレンドを抽出。
2.  **企画**: AI がトレンドに基づき、ブログ記事や SNS 投稿のネタを 30個生成。
3.  **制作**: n8n 内で AI が自動で下書きを執筆。
4.  **配信**: Metricool を通じて X, Instagram, TikTok 等へ予約投稿。
5.  **分析**: Metricool の統計を見て、反応の良いネタへ翌日のリサーチを絞る。

## 3. 24時間稼働ロードマップ (インフラ戦略)
「技術習得」と「低コスト・高安定性」を両立させるための推奨ステップです。

1.  **Phase 1: VPS サーバー (今すぐ)**
    *   **ターゲット**: DigitalOcean や さくらのVPS ($6〜10/月)。
    *   **目的**: 24時間安定稼働する自動化基盤の即時構築。Docker を使い、設定ミスで PC が壊れるリスクをゼロにします。
2.  **Phase 2: API 安全装置の設置**
    *   **予算制限 (Hard Limit)**: 各 API 管理画面で「月額 $10 以上は自動停止」を設定。
    *   **プリペイド**: xAI や OpenAI に事前に入金した分だけを使うよう設定し、物理的に高額請求を防ぎます。
3.  **Phase 3: 自宅ミニPCサーバー (将来)**
    *   **ターゲット**: Mac mini (M1/M2) または ミニPC (N100 搭載機など)。
    *   **目的**: 月額のサーバー代すら 0円に近づけます。Mac mini は消費電力が極めて低く、24時間稼働に最適です。
4.  **Phase 4: ローカル AI (LLM) の導入**
    *   **目的**: API を叩かずに、自宅サーバー内の AI で処理を完結させ、API コストを完全撤廃します。

---

## 4. セキュリティ ＆ 安全ガイドライン
*   **API キーのリソース管理**: 
    *   API サービス側のダッシュボードで、必ず **Usage Limits (予算制限)** を設定してください。
*   **サーバー保護**: 
    *   外部から n8n 等へアクセスする場合、Cloudflare Tunnel 等の暗号化されたトンネル技術を使用し、自宅のネットワークを直接ネットに晒さないようにします。
*   **バックアップ**: 
    *   作成したワークフロー（JSON）は、定期的に `/logs/backups/` フォルダへ Git 管理して保管してください。
