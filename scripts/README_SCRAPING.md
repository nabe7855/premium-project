# ホテルデータスクレイピング引き継ぎ書

このプロジェクトでは、Couples.jpからホテル情報を取得し、データベースに保存するためのスクリプト群を用意しています。
情報の鮮度を保つため、いつでも実行可能です。

## 事前準備

Node.jsがインストールされている環境で、以下のライブラリが必要です（インストール済み）。

```bash
npm install cheerio @prisma/client uuid
```

## 実行手順

スクレイピングは2つのフェーズに分かれています。

### フェーズ1: リンク作成（IDマッピング）

データベース内に既にあるホテルの「名前」や「電話番号」を元に、Couples.jp上の詳細ページIDを取得し、`place_id` カラムに保存します。

```bash
node scripts/scrape_phase1_map_ids.cjs
```

- **目的**: 新しいホテルが追加された際や、リンクが切れている場合に実行します。
- **処理**: 47都道府県の一覧ページをスキャンし、一致するものがあればIDを紐付けます。

### フェーズ2: 詳細情報の取得

IDが紐付いているホテルに対して、詳細ページ（料金、アメニティ、公式サイトURL等）をスクレイピングしてDBを更新します。

```bash
node scripts/scrape_phase2_details.cjs
```

- **目的**: 料金の更新、不足している設備情報や公式サイトURLの取得。
- **取得項目**:
  - `website` (公式サイトURL)
  - `distance_from_station` (最寄り駅・徒歩)
  - `room_count` (部屋数)
  - `min_price_rest` / `min_price_stay` (最安値)
  - `price_details` (詳細な料金表 - JSON形式)
  - `amenities` / `services` (アメニティ・設備・サービス項目)

## 注意事項

- **実行間隔**: サーバーに負荷をかけないよう、各リクエスト間に待機時間（sleep）を設けています。大量のデータを一気に取得するとブロックされる可能性があるため、時間はかかりますが安全に実行されます。
- **データの重複**: 名前と電話番号で既存データを確認しているため、重複登録はされません。
- **画像の著作権**: 現在、画像ファイル自体のダウンロードは行わず、URLの取得にとどめています（法的リスク回避のため）。画像が必要な場合は、別途Google Places APIなどの正規ルートでの取得を推奨します。
