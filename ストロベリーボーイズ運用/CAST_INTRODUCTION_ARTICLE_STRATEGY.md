# 新人キャスト入店時 紹介記事 制作戦略

> **目的**：新人キャスト入店のタイミング（＝最もホットな瞬間）を活用し、SEO・AIO両面で効果の高いボリューミーな紹介記事を制作する
> **対象**：記事制作を担当する全スタッフ（専門知識不要で運用できるフォーマット）
> **位置づけ**：SNS × SEO 総合アクションプランにおける「店舗ページ × コンテンツマーケティング」施策

---

## 1. なぜ「入店時」が最重要タイミングなのか

入店直後は、キャストにとっても店舗にとっても最大のチャンスである。理由は3つある。

**鮮度のシグナル**：Googleは新しいコンテンツを「フレッシュネスシグナル」として評価する。新人入店というイベントは自然に新規コンテンツを生み出す口実になり、サイト全体の更新頻度を上げることでクロール頻度の向上にもつながる。

**ユーザーの関心が最も高い瞬間**：「新しいキャストが入った」という情報は既存のお客様にとって最も興味を引くニュースであり、SNSでの拡散力・クリック率が最高になるタイミングである。

**キャスト本人のモチベーション**：入店時はキャスト自身も積極的に情報提供・写真撮影に協力してくれる。時間が経つと協力を得にくくなるため、このタイミングでリッチなコンテンツを作り切ることが重要。

## 2. 記事が狙うSEO・AIO効果

### 2.1 ターゲットキーワード設計

1つの記事で複数のキーワードクラスターを狙う。

**メインKW（記事タイトル・H1に含める）**
- 「ストロベリーボーイズ ＋ キャスト名」（指名検索の受け皿）
- 「女性用風俗 横浜（or 福岡） キャスト紹介」

**サブKW（見出し・本文に自然に散りばめる）**
- 「女性用風俗 おすすめ キャスト」
- 「出張ホスト 横浜（or 福岡）」
- 「女性 癒やし マッサージ 横浜（or 福岡）」
- 「ストロベリーボーイズ 口コミ」
- 「女性用風俗 初めて」

**ロングテールKW（記事内のQ&Aセクションで拾う）**
- 「女性用風俗 どんな人が来る」
- 「女性用風俗 キャスト 選び方」
- 「ストロベリーボーイズ ＋ キャスト名 ＋ 口コミ」

### 2.2 AIO（AI Overview）最適化

Google AI OverviewやChatGPT検索に引用されるためには、「簡潔で構造化された回答」がページ内に存在する必要がある。以下の要素を必ず記事に含める。

- **FAQ形式のセクション**：「Q: このキャストはどんな方ですか？」→「A: 〇〇は…」の形式。AI Overviewが引用しやすい構造。
- **箇条書きのプロフィールサマリー**：名前・身長・趣味・得意なことを箇条書きで冒頭に配置。AIがエンティティ情報として認識しやすい。
- **「〇〇とは」の定義文**：記事冒頭に「〇〇（キャスト名）は、ストロベリーボーイズ横浜店（or 福岡店）に所属する…」という定義文を置く。これはAIOが最も引用しやすいパターン。
- **構造化データ（JSON-LD）**：Person schema + FAQPage schemaを記事ページに埋め込む（開発チーム連携）。

## 4. 記事構成テンプレート（推奨3,000〜5,000字）

以下のテンプレートに沿って記事を構成する。各セクションの順番には意味があり、SEO・ユーザー体験の両面で最適化されている。

### 記事構成

```
【タイトル】
{キャスト名}｜ストロベリーボーイズ{店舗名}店の新人キャストをご紹介
※40〜60文字。キャスト名 + ブランド名 + 地域名を必ず含める

【meta description】
{キャスト名}はストロベリーボーイズ{店舗名}店に新しく入店したキャストです。
{一言の魅力}。プロフィール・得意なサービス・デートプランまで詳しくご紹介します。
※120〜155文字。クリック率を左右する最重要要素。

---

■ セクション1：リード文（200〜300字）
- キャスト名 + 所属店舗 + 一行の定義文（AIO対策）
  「{キャスト名}は、ストロベリーボーイズ{店舗名}店に新しく加わったキャストです。」
- どんな方に特におすすめか（ターゲット明示）
- 記事で何がわかるか（読了のメリット提示）

■ セクション2：プロフィールカード（箇条書き）
- 源氏名
- 所属店舗
- 身長
- 年齢層
- 趣味
- 得意なサービス
- キャッチコピー（一言で表す魅力）
※構造化データ（Person schema）で機械的にも読み取れるようにする

■ セクション3：{キャスト名}ってどんな人？（800〜1,200字）
- パーソナリティの紹介（質問7〜12の回答を元に物語的に構成）
- 「休日の過ごし方」「ハマっていること」で人柄を立体的に描く
- 友人や周囲からの評価（第三者の視点）を交えて信頼感を出す
- 適宜、写真（素材C）を挟んで視覚的にも楽しませる

■ セクション4：お仕事への想い（600〜800字）
- この仕事を始めたきっかけ（ストーリー性で読み物としての吸引力）
- お客様にどんな時間を届けたいか（サービスの価値訴求）
- 接客で大切にしていること（信頼感・安心感の醸成）
- もしあれば、お客様からの嬉しかった言葉

■ セクション5：{キャスト名}の得意なサービス（400〜600字）
- 得意な施術・サービスの具体的な説明
- どんなお客様に特に喜ばれるか
- 他のキャストとの差別化ポイント
※「女性用風俗 おすすめ キャスト」等のKWを自然に含める

■ セクション6：{キャスト名}とのおすすめデートプラン（400〜600字）【任意】
- キャスト本人が語る理想のデートプラン
- おすすめエリア・スポットの紹介
- SweetStayの関連記事への内部リンク
  「{エリア名}のおすすめホテルはこちら → [SweetStay記事リンク]」
※地域系KW + SweetStay連動で内部リンク構造を強化

■ セクション7：初めてのお客様へのメッセージ（200〜300字）
- キャスト本人の言葉をそのまま掲載（温度感を大切に）
- 「初めてで不安な方にも…」の一文で初心者KWを自然に拾う
- 予約導線（CTA）への自然な橋渡し

■ セクション8：よくある質問（FAQ）（400〜600字）
- Q1: {キャスト名}さんはどんなタイプの方におすすめですか？
- Q2: 初めてでも大丈夫ですか？
- Q3: 指名予約はできますか？
- Q4: デートプランの相談はできますか？
※FAQPage schemaで構造化。AIO引用率を最大化する最重要セクション。

■ セクション9：予約・お問い合わせ（CTA）
- 予約ボタン（明確なCTA）
- 電話番号・LINE等の問い合わせ導線
- 「まずはお気軽にご相談ください」の一文
```

### 記事の目標ボリューム

| 項目 | 目安 |
|------|------|
| 総文字数 | 3,000〜5,000字 |
| 画像枚数 | 5〜8枚 |

## 5. ライティングガイドライン

### 5.1 文体・トーン

- **基本トーン**：温かく、上品で、親しみやすい。雑誌のインタビュー記事のような読み心地を目指す。
- **NGトーン**：事務的な箇条書きだけの記事、過度にカジュアルすぎる言葉遣い、直接的な性的表現。
- **一人称**：キャスト本人の言葉は「」で囲み、地の文は三人称で書く。「〇〇さんは、笑顔でそう語る。」のようなインタビュー記事形式が理想。

### 5.2 SEOライティングのルール

**タイトルタグ（title）**
- 40〜60文字以内
- 「{キャスト名}｜ストロベリーボーイズ{店舗名}店の新人キャストをご紹介」
- キャスト名を先頭に置く（指名検索で最も目立つ位置）

**見出し（H2・H3）にキーワードを含める**
- H2：「{キャスト名}ってどんな人？」「{キャスト名}の得意なサービス」
- 見出しだけ読んでも記事の概要が分かるようにする

**内部リンクの設置ルール**
- 最低3本：①SweetStay関連記事、②他のキャスト紹介記事、③サービス・料金ページ
- アンカーテキストにはKWを含める
  - 良い例：「横浜エリアのおすすめホテルはこちら」
  - 悪い例：「こちらをクリック」

**画像のalt属性**
- 全画像にalt属性を必ず設定する
- 例：「ストロベリーボーイズ横浜店 キャスト〇〇 プロフィール写真」
- キーワードを自然に含めつつ、画像の内容を正確に説明する

### 5.3 表現ガイドライン（コンプライアンス）

SNS × SEO 総合アクションプランの表現ガイドラインを踏襲する。

**使ってよい表現**
- 女性専用リラクゼーション、出張ホスト、癒やしサービス
- パートナーシップサポート、セルフケア、ウェルネス
- 「女性のための」「あなたのための」

**使ってはいけない表現**
- 直接的な性的表現・ワード
- 「風俗」の単語はSEO記事本文内では使用可だが、過度な繰り返しは避ける
- 他店との比較や誹謗中傷

## 6. 構造化データ（テクニカルSEO）

開発チームと連携し、各キャスト紹介記事に以下の構造化データを埋め込む。

### 6.1 Person Schema（キャストのエンティティ認識）

```json
{
  "@context": "https://schema.org",
  "@type": "Person",
  "name": "{キャスト名}",
  "jobTitle": "キャスト",
  "worksFor": {
    "@type": "Organization",
    "name": "ストロベリーボーイズ",
    "url": "https://strawberry-boys.com"
  },
  "workLocation": {
    "@type": "Place",
    "name": "ストロベリーボーイズ {店舗名}店",
    "address": {
      "@type": "PostalAddress",
      "addressLocality": "{横浜 or 福岡}"
    }
  },
  "image": "{プロフィール画像URL}",
  "description": "{キャスト紹介の要約文}"
}
```

### 6.2 FAQPage Schema（AIO引用対策）

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "{質問文}",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "{回答文}"
      }
    }
  ]
}
```

### 6.3 BreadcrumbList Schema（サイト構造の明示）

```json
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "TOP", "item": "https://strawberry-boys.com" },
    { "@type": "ListItem", "position": 2, "name": "{店舗名}店", "item": "https://strawberry-boys.com/{store-slug}" },
    { "@type": "ListItem", "position": 3, "name": "キャスト一覧", "item": "https://strawberry-boys.com/{store-slug}/casts" },
    { "@type": "ListItem", "position": 4, "name": "{キャスト名}", "item": "https://strawberry-boys.com/{store-slug}/casts/{cast-slug}" }
  ]
}
```

## 8. 内部リンク戦略（サイト全体のSEOを強化）

キャスト紹介記事は「孤立したページ」にしてはならない。サイト全体のSEO力を上げるために、以下の内部リンク設計を徹底する。

### 8.1 キャスト紹介記事 → 他ページへのリンク

- **SweetStay記事**：キャストのおすすめエリアに関連するホテル記事へリンク
- **イケジョラボ記事**：「初めての女性用風俗」系の教育記事へリンク
- **他キャスト紹介記事**：「他のキャストも見る」セクションを設ける
- **サービス・料金ページ**：CTAとして自然にリンク
- **店舗ページ**：所属店舗の詳細ページへリンク

### 8.2 他ページ → キャスト紹介記事へのリンク

新しいキャスト記事を公開したら、以下の既存ページにもリンクを追加する。

- **キャスト一覧ページ**：新キャストのカードを追加
- **店舗トップページ**：「新着キャスト」セクションに掲載
- **関連するSweetStay記事**：「このエリアが好きなキャスト」として紹介
- **SNSからの導線**：全プラットフォームのプロフィールリンクを最新記事に更新

### 8.3 内部リンクのアンカーテキスト例

| リンク先 | アンカーテキスト例 |

### 9.2 改善サイクル

**公開後1週間**：Search Consoleでインデックス状況を確認。インデックスされていなければ手動でリクエスト。

**公開後2週間**：SNS経由の流入数を確認。想定より少なければ、SNS投稿のクリエイティブ（画像・コピー）を変えて再投稿。

**公開後1ヶ月**：GA4で滞在時間・離脱率を確認。離脱率が高いセクションがあれば、そのセクションの内容を改善。

**公開後3ヶ月**：「キャスト名」での検索順位を確認。1位でなければタイトル・descriptionを微調整。FAQ内容を追加・更新。

## 付録：記事タイトル例

以下はキャスト名を仮に設定した場合のタイトル例。バリエーションを参考に、実際のキャスト名で最適なタイトルを選ぶ。

1. **{ハルト}｜ストロベリーボーイズ横浜店の新人キャストをご紹介**
   - 最もスタンダード。指名検索＋ブランド名＋地域名を確実にカバー。

2. **【新人紹介】{ハルト} - ストロベリーボーイズ福岡店｜癒やし系キャストの魅力に迫る**
   - 「新人紹介」のラベルで新鮮さを演出。やや長めだがクリック率向上が期待できる。

3. **{ハルト}ってどんな人？ストロベリーボーイズ横浜のNEWキャストを徹底紹介**
   - 「〇〇ってどんな人？」は検索クエリに一致しやすく、AIO引用の可能性が高いパターン。
