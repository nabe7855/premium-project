## 1. インタビュー素材分析 — サイの「金脈」

トランスクリプトを通して読んだ結果、サイさんの素材としてのポテンシャルは非常に高い。以下がSEO的にも読み物としても活きるポイント。

### ◎ 記事の軸になる「核」

**「寄り添いの人」**がサイさんの一貫したテーマ。源氏名の由来（礼拝の「際」＝人に寄り添う）、本職の福祉、接客スタイル、植物の世話、すべてが「そっと横にいる人」という人物像に収束する。これは記事のトーンそのものになる。

### ◎ SEO的に強い差別化要素

- **糸島 × デートプラン**: 「糸島 デート」は福岡エリアで検索ボリュームのあるキーワード。卵プリン、海鮮、ドライブといった具体的な地元情報が出てくるため、ローカルSEOとの相乗効果が見込める
- **オイルマッサージの実力**: 「女性用風俗 マッサージ」系のロングテールに刺さる。セラピスト経験者という裏付けがある
- **初心者向け**: 「女性用風俗 初めて」は最重要キーワードの一つ。サイさんの「緊張をほぐす雰囲気」「物腰の柔らかさ」は初心者訴求に最適

### ◎ 読み物として面白い「ギャップ」

- 普段はおっとり天然 → WANIMAのライブではジャンプして暴れる
- 松山ケンイチ似の犬っぽい動き → 塩顔でくっきりした目元
- 福祉職の真面目さ → 「ちょっと闇を感じる」とインタビュアーに言わせる中毒性
- 植物を静かに見守る日常 → キャンプでアウトドアを満喫

## 2. 記事構成案 — サイ編

### タイトル案（SEO + 興味引き）
```
「おっとり塩顔セラピスト・サイに聞いた、植物と煮物とWANIMAの話」
```
- 「塩顔」「セラピスト」→ 検索キーワード
- 「植物と煮物とWANIMA」→ 意外性で興味喚起。3つ並べることでリズムが出る

### セクション構成

1. **導入**（300字）: 第一印象 — 画面越しでも伝わるゆるさ、声のトーン、「この人に緊張する方が難しい」という空気感
2. **名前の由来**（300字）: 「際」の意味、以前のセラピスト経験、名前に込めた哲学
3. **地元・糸島トーク**（500字）: 海と山、卵プリン、ドライブコース。デートプランとしての糸島の魅力。「このエリアならお任せください」
4. **見た目と雰囲気**（400字）: 松山ケンイチ似、塩顔、まつ毛、ネルシャツ。「下北沢にいそう」
5. **日常の過ごし方**（400字）: 植物の世話、ぼーっとする時間の大切さ、カフェ探索。ここに人柄が滲む
6. **ギャップ — WANIMAと暗い漫画**（300字）: おっとりの裏側。ライブで暴れる一面、CLAMPSのような闇のある作品も好き
7. **仕事への姿勢**（400字）: 寄り添う力、共感型、所作への気配り、オイルマッサージへの自信
8. **初めての方へ**（300字）: 本人メッセージ。ホテル選びのアドバイス（ワの紹介）
9. **FAQ**（3問）: AIO最適化用。構造化データ対応

### 想定文字数：3,000〜3,500字

## 3.「イトウ」— 架空ライター/インタビュアー構想

### 結論：これはかなりいいアイデア

理由を3つ挙げる。

### ① AI検知回避 × 人間味の注入

純粋にQ&A形式で記事を生成すると、どうしてもLLMの癖（均一な文体、予測可能な展開、きれいすぎる構成）が出る。しかし「イトウ」というキャラクターのフィルターを通すことで、脱線、ツッコミ、個人的な感想が自然に入る。これはAI検知のシグナルを大幅に下げる。

### ② コンテンツのシリーズ化

「イトウが行く！キャスト丸裸インタビュー」のようなシリーズタイトルにすれば、記事間の回遊が生まれる。読者が「イトウの次のインタビューも読みたい」と思えば、サイト滞在時間が伸びてSEOにも効く。

### ③ インタビュアーの存在が「安心感」になる

女性向け風俗は、利用者にとって情報収集段階でかなり心理的ハードルがある。「店の中の人（イトウ）がフランクに話を聞いている」という構図は、読者を「傍聴者」の立場に置く。直接キャストの自己紹介を読むより、第三者が間に入っている方が安心して読める。

### イトウのキャラクター設計案

| 項目 | 設定 |
|------|------|
| 名前 | イトウ（下の名前はあえて出さない。親しみやすさ重視） |
| 役職 | 女性向け風俗ライター（業界全般に精通したフリーランス的ポジション） |
| 性格 | ちょっとおっちょこちょい、好奇心旺盛、脱線しがち、でも根は真面目。キャストに対して「僕が個人的に気になる」スタンスで聞く |
| 口調 | 丁寧語ベースだけどフランク。「〜なんですよね」「いや、それめっちゃいいですね」「ちょっと待って、僕が行きたくなってきた」 |
| 特徴的な行動 | 自分の話をちょいちょい挟む（「僕は植物すぐ枯らすタイプなんで…」）、素のリアクションが大きい、たまに質問を忘れる |
| 決め台詞 | 冒頭：「今回もキャストのことを丸裸にしていきます！」 |
| ビジュアル | アイコンのみ（シルエットやイラスト）。実在しない設定なので顔出し不要 |

### 重要な設計ポイント

**イトウ ≒ もりさん自身をデフォルメしたキャラ**にするのがベスト。実際のインタビューで「僕が福岡に興味ありすぎて時間使い切りそう」とか「僕が個人的に行きたいな」という自然なリアクションがトランスクリプトに出ている。この「素の好奇心」がそのままイトウの武器になる。

ただし、もりさん本人と完全に同一視されないよう、あくまで「店舗スタッフのイトウ」という一歩引いた位置にする。これにより、もりさんが運営者としての立場を保ちつつ、コンテンツ上では「イトウ」が自由に動ける。

## 5. 運用フロー

```
インタビュー音声
    ↓
文字起こし（Whisper API）
    ↓
構造化JSON生成（Claude API — profile_structured.json）
    ↓
「イトウ」視点の記事ドラフト生成（Claude API）
    ├─ プロンプトにイトウのキャラ設定を常に含める
    ├─ インタビュー中のインタビュアー発言も素材として活用
    └─ 「イトウの感想・脱線」を意図的に挿入する指示
    ↓
人間レビュー（もりさん）
    ↓
公開 + SNS展開
```

### LLMプロンプト設計のポイント

記事生成時のプロンプトには以下を常に含める：

- イトウのキャラクター設定（口調、性格、行動パターン）
- 「完璧な文章にしないこと」という明示的指示
- インタビュー中のインタビュアー（もりさん）の実際のリアクションをイトウの台詞として再構成
- 「1箇所は脱線すること」「1箇所はイトウの個人的エピソードを入れること」
