# ホテルサイト（Sweet Stay）開発進捗 & 引き継ぎガイド

本ドキュメントは、ホテルポータルサイト「Sweet Stay」の開発進捗、データベース構造、および今後のタスクをまとめたものです。

## 1. プロジェクト概要

全国のラブホテル情報を網羅し、AIを活用した口コミ生成とリッチなコンテンツ提供を行う次世代ホテルポータル。

## 3. データベース構造（主要テーブル）

- `lh_hotels`: ホテルの基本情報、料金、AI説明文、スクレイピング済み断片（`review_snippets`）。
- `lh_reviews`: AI生成されたオリジナル口コミ（`is_verified`, `is_cast` フラグあり）。
- `lh_amenities` / `lh_hotel_amenities`: 設備・備品マスターとその紐付け。
- `lh_services` / `lh_hotel_services`: サービス（デリバリー可、外出可など）のマスタ。
- `lh_feature_articles`: エリア別・機能別特集記事。

## 4. 実行済み・利用可能なスクリプト (scripts/ 配下)

- `scrape_hotel_data.cjs`: Phase 1（名称、住所、電話番号の取得）。
- `scrape_phase2_details.cjs`: Phase 2（料金、アメニティ、レビュー断片の深掘り取得）。
- `tmp/check_hotel_progress.ts`: 現在の進捗（カウント）を確認する用。

## 5. 今後のタスクロードマップ（引き継ぎ用）

### TASK A: データ補完・クレンジング

1.  **Phase 2 Scraping の完遂**:
    - `place_id` を持ちながら `website` が未登録（未処理）の約2,000件に対し、詳細スクレイピングを実行。
    - 実行コマンド案: `node scripts/scrape_phase2_details.cjs`
2.  **料金データの正規化**:
    - `price_details` (JSON) から「平日休憩最安値」等を取り出し、ソート・表示用のフィールドへ展開。

### TASK B: AIコンテンツ錬成（コスト0円戦略）

1.  **AI口コミ生成の実装**:
    - `zero_cost_ai_reviews_plan.md` に基づき、`review_snippets` を Gemini API (Flash) でリライト。
    - **ルール**: 他サイトのコピペは厳禁（法的・SEOリスク回避）。事実の断片から、複数のペルソナ（20代女子会、出張利用等）で新規書き起こし。
2.  **AI説明文・キャッチコピー生成**:
    - `raw_description` を元に、1ホテルごとにSEOに強い紹介文を生成し `ai_description` に保存。

### TASK C: UI/UX & メディア戦略

1.  **店舗詳細ページの完成**:
    - `src/app/(protected)/sweetstay/hotel/[id]` において、取得したアメニティ（アイコン付）やAI口コミを美しく表示。
2.  **エリア別特集の自動生成**:
    - 取得した「設備タグ（露天風呂、TVサイズ等）」とエリアを組み合わせ、特集記事を自動生成。

## 6. 開発再開時の注意点

- **API制限**: Gemini APIを利用する際は、無料枠のレート制限（1分15回/1日1500回等）を考慮したスロットリング（4-5秒に1回実行）を徹底すること。
- **重複チェック**: スクレイピング時は常に名称または電話番号での `findFirst` を行い、重複登録を避けること。
