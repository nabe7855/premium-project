# ハブページ（/test13）およびサイト全体構造の改修計画

本ドキュメントは、ストロベリーボーイズ（または対象となる類似サービスサイト）のハブページ（`/test13`等）全体のUI/UX、SEO/AIO、および情報アーキテクチャ（IA）の根本的な改修計画をまとめた引き継ぎ用資料です。
他のAIや開発メンバーが直ちに実装へ移行できるよう、要件定義と設計方針を構造化して記載しています。

---

## 1. プロジェクトの背景と現状の課題

現状のサイト（特に `/test13` などのハブページ）には、以下の主要な課題が存在します。

1. **サイト構造・IAの不明瞭さ**
   レイアウトが複雑で、どの情報にアクセスすべきかユーザーが迷いやすい。階層構造が未整理で内部リンクが不足しているため、検索エンジンからのクロール効率（SEO）も低下している。
2. **UXの課題（回遊率・CV導線）**
   ファーストビューに検索バーや明確なCTA（予約等）が不足。Netflixのような一目でわかるシンプルな階層がなく、「動画視聴 → 関連記事 → キャスト詳細 → 店舗 → 予約」という理想的なカスタマージャーニー（回遊導線）が分断されているため、直帰率が高まりやすい。
3. **SEOの課題（トピッククラスター未形成）**
   「人気キャスト」「初心者ガイド」「店舗一覧」などのピラーページ（テーマ別のまとめページ）が未整備。関連ページ同士のつながりが弱く、キーワード（例：女性用風俗 初心者）での順位獲得の機会を逃している。
4. **コンテンツの乱立**
   「検索（Tabelog型）」「エンタメ（Netflix型）」「教育（オウンドメディア型）」の3要素が統合されておらず、情報が散在している。

## 2. 改修のコア戦略（SEO・AIO・UX）

### 2.1 UI/UXの改善（Netflix & Tabelog型ハイブリッド）

- **ファーストビューの再設計**: メインビジュアル（動画や大型バナー）で直感的に魅力を伝え、同時に「キャストを探す」「店舗から探す」など明確な1つのCTA（検索窓等）を配置する。
- **カード式・グリッドレイアウト**: ユーザーが直感的に選べるよう、コンテンツ（人気キャスト、おすすめ動画、初心者ガイド等）をカテゴリごとにカード化して並べる。
- **シームレスな回遊導線**: `動画を見た後` -> `関連記事を表示` -> `関連キャストへ誘導` -> `所属店舗を確認` -> `予約（CV）` というストーリー性のある導線をページ・コンポーネント間に組み込む。

### 2.2 情報アーキテクチャ（IA）とSEO改善

- **ピラミッド型階層構造**: `トップページ` -> `主要カテゴリ（ピラーページ）` -> `詳細ページ（キャスト/店舗/記事）` へと展開する一本化されたツリー構造を構成する。
- **カテゴリの再編成（ピラーページ化）**: 「地域別店舗」「初心者向けガイド」「人気ランキング」「セラピスト特集」など、ターゲットキーワードを意識したテーマ分けを行う。
- **トピッククラスターと内部リンクの強化**: ピラーコンテンツ（例：初心者ガイド）を中心に、各詳細ページへのコンテキストリンク（文脈に沿ったリンク）を多用。ページ間に関連性を持たせ、リンクジュースを最適に循環させる。

### 2.3 AIO（AI検索最適化）の導入

- **FAQ構造の追加**: 「初めての予約方法」「料金システム」など、ユーザーの疑問と回答をまとめ、FAQスキーマ（Structured Data）を実装する。
- **まとめページ（ピラー）の強化**: AIが「信頼できる網羅的な情報源」として引用しやすいよう、各種テーマに関する要点を集約したページを作成する。
- **構造化データの徹底**: `Organization` / `LocalBusiness`（店舗）、`FAQPage`（Q&A）、`VideoObject`（動画）のスキーマを適切にマークアップし、AI検索エンジンへの文脈理解を促進する。

## 3. ハブページ (`/test13`) の理想レイアウト設計

`/test13` は、「検索」「エンタメ」「教育」の3要素を統合した全サイトの中核ハブとして機能させます。

1. **【上部】 検索・フィルターエリア**
   - 巨大で目立つ検索バーを配置。
   - 「キャスト名」「店舗名」「フリーワード」での直接検索機能。
   - 位置情報やカテゴリによる絞り込み機能（Tabelog的アプローチ）。
2. **【中央】 エンタメ・ギャラリーセクション**
   - 新着/特集動画を大きなサムネイルでカルーセル表示（自動再生考慮）。
   - 「注目CAST」「カテゴリ別おすすめCAST」などをNetflixのようなカード式で横スクロール表示。
3. **【下部】 教育・ガイドコンテンツセクション**
   - 「初めてガイド」「よくある質問（FAQ）」「利用術（コラム）」などの見出しを立て、初心者向けの記事リンクを整理して提示。
4. **【サイド/フッター】 ランキング・CTA**
   - 「人気CASTランキング」「店舗ランキング」等への導線。
   - 常にアクセス可能な「今すぐ予約」や「LINE登録」の固定（追従）ボタン。

## 5. 今後の実装ステップ（別のAI・開発者向け）

この計画書を元に開発を進める際の手順案です。

1. **データスキーマ・構造化データの定義**:
   - `LocalBusiness`, `FAQPage`, `VideoObject` などをNext.jsのMetadata・JSON-LDとして組み込む準備等の確認。
2. **ハブページ（`/test13`）のUIコンポーネント実装**:
   - `HeroSection`（巨大検索窓付き）
   - `EntertainmentCarousel`（動画・キャストカード表示）
   - `EducationSection`（ガイド・FAQ表示）
3. **ルーティングとナビゲーションの整備**:
   - 上記サイトマップに基づく、各ピラーページ（`/casts`, `/stores`, `/guides`）の作成とパンくずリストの実装。
4. **内部リンク最適化**:
   - 詳細ページ内の下部やサイドバーに、関連コンテンツを呼び出すコンポーネント（例：`RelatedCasts`, `RelatedArticles`）を実装・配置。

この資料（`HUB_PAGE_RENOVATION_PLAN.md`）を読み込ませることで、どのAIアシスタントでも現在の改修フェーズと次に行うべきコーディングタスクを把握できるように設計・構成されています。
